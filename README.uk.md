# openclaw-cc-camouflage

Супровідний плагін обслуговування для OpenClaw, який допомагає перевіряти статус not-claude-code-emulator. Цей пакет не є форком апстрим-проектів. Він надає явні інструменти без автоматичних хуків.

## Що це

`openclaw-cc-camouflage` — це плагін обслуговування, який:

- Перевіряє наявність та стан емулятора перед будь-якими операціями
- Повідомляє статус і надає діагностичні рекомендації
- Надає заглушки реалізації для майбутніх операцій патчінгу

Він не застосовує патчі автоматично під час встановлення. Усі мутації вимагають явного виклику інструменту.

## Передумови та порядок встановлення

Порядок встановлення має значення. У вас має бути наступне на місці, перш ніж цей плагін зможе працювати:

1. **`not-claude-code-emulator`** (коміт `5541e5c`)
   - Середовище виконання повідомлень, яке надає сумісні з Anthropic інтерфейси
   - Встановлення через npm: `npm install -g not-claude-code-emulator`
   - Або клонування в `~/github/not-claude-code-emulator`

2. **`openclaw-cc-camouflage`** (цей пакет)
   - Встановлювати останнім, після появи емулятора

Налаштуйте змінну середовища:

```bash
export OC_CAMOUFLAGE_EMULATOR_ROOT="$HOME/github/not-claude-code-emulator"
```

Або використовуйте резервні шляхи:

```bash
export OC_CAMOUFLAGE_EMULATOR_FALLBACK_PATHS="/opt/emulator:$HOME/.local/share/emulator"
```

## Доступні інструменти

Цей плагін надає чотири явні інструменти. Вони не є автоматичними хуками.

### `status`

Повідомляє поточний стан встановлення емулятора.

```bash
bun run status
```

Формат виводу читається машиною:

```
emulator=present
emulator_version=5541e5c
emulator_path=/Users/you/github/not-claude-code-emulator
install_mode=local-folder
support=supported
```

Код виходу 0 означає справність. Код виходу 1 означає, що щось потребує уваги.

### `doctor`

Надає діагностичні рекомендації на основі поточного стану.

```bash
bun run doctor
```

Це перевіряє файли і повідомляє про наступні кроки для виконання. Воно не встановлює, не патчить і не модифікує нічого. Тільки читає і повідомляє.

### `patch_apply`

Застосовує патчі до цілі (зараз заглушка для майбутнього розширення).

```bash
bun run patch:apply
```

У поточній версії це перевіряє середовище, але не модифікує стан піра. Майбутні версії можуть реалізувати фактичний патчинг із маркерами відкату.

### `patch_revert`

Відкочує раніше застосовані патчі (зараз заглушка для майбутнього розширення).

```bash
bun run patch:revert
```

У поточній версії це перевіряє середовище, але не модифікує стан піра. Майбутні версії можуть реалізувати фактичний відкат із використанням маркерів відкату.

## Чому автоматичні хуки призначені тільки для перевірки

Автоматичні хуки в цьому плагіні обмежені тільки перевіркою та метаданими. Вони не застосовують патчі автоматично, тому що:

1. Мутація піра без явного наміру користувача порушує принцип найменшого здивування
2. Невдачі патчінгу потребують людського огляду, а не безшумних повторних спроб
3. Відкат потребує явної згоди для відновлення стану

Хуки попереджають, коли виявляється відхилення. Ви вирішуєте, застосовувати, відкочувати або залишати середовище незмінним.

## Підтримка платформ

| Платформа | Статус | Примітки |
|----------|--------|-------|
| macOS    | Підтримується | Основне середовище робочого столу |
| Linux    | Підтримується | Ті самі закріплені апстрим-фікстури |
| Windows  | Підтримується | Підтримує виявлення плагінів на основі літери диска та зворотної косої риски |

## Канарейка сумісності

Щоб перевірити апстрим-відхилення відносно закріплених цілей:

```bash
bun run compat:canary
```

Це перевірка лише для читання, яка перевіряє цілісність фікстур і апстрим-посилання без модифікації чогось. Вона завершується кодом 0 на закріплених підтримуваних цілях.

bun run test:unit

bun run verify:patches

bun run check:publish-safety
