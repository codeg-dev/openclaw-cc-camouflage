# openclaw-cc-camouflage

OpenClaw এর জন্য একটি সঙ্গী রক্ষণাবেক্ষণ প্লাগইন যা not-claude-code-emulator স্থিতি যাচাই করতে সহায়তা করে। এই প্যাকেজটি আপস্ট্রিম প্রকল্পের একটি ফর্ক নয়। এটি স্বয়ংক্রিয় হুক ছাড়াই স্পষ্ট টুলিং প্রদান করে।

## এটি কী

`openclaw-cc-camouflage` একটি রক্ষণাবেক্ষণ প্লাগইন যা:

- যেকোনো অপারেশনের আগে এমুলেটারের উপস্থিতি এবং স্বাস্থ্য যাচাই করে
- স্থিতি রিপোর্ট করে এবং ডায়াগনোস্টিক নির্দেশনা প্রদান করে
- ভবিষ্যতের প্যাচ অপারেশনের জন্য স্টাব বাস্তবায়ন প্রদান করে

এটি ইনস্টলের সময় স্বয়ংক্রিয়ভাবে প্যাচ করে না। সমস্ত পরিবর্তনের জন্য স্পষ্ট টুল আহ্বান প্রয়োজন।

## পূর্বশর্ত এবং ইনস্টল অর্ডার

ইনস্টল অর্ডার গুরুত্বপূর্ণ। এই প্লাগইন কাজ করার আগে আপনার কাছে নিম্নলিখিত থাকতে হবে:

1. **`not-claude-code-emulator`** (কমিট `5541e5c`)
   - বার্তা রানটাইম যা Anthropic-সংগত ইন্টারফেস প্রদান করে
   - npm এর মাধ্যমে ইনস্টল করুন: `npm install -g not-claude-code-emulator`
   - অথবা `~/github/not-claude-code-emulator` তে ক্লোন করুন

2. **`openclaw-cc-camouflage`** (এই প্যাকেজ)
   - এমুলেটার উপস্থিত হওয়ার পরে সবশেষে ইনস্টল করুন

পরিবেশ ভেরিয়েবল কনফিগার করুন:

```bash
export OC_CAMOUFLAGE_EMULATOR_ROOT="$HOME/github/not-claude-code-emulator"
```

অথবা ফলব্যাক পথ ব্যবহার করুন:

```bash
export OC_CAMOUFLAGE_EMULATOR_FALLBACK_PATHS="/opt/emulator:$HOME/.local/share/emulator"
```

## উপলব্ধ টুলস

এই প্লাগইনটি চারটি স্পষ্ট টুল এক্সপোজ করে। এগুলো স্বয়ংক্রিয় হুক নয়।

### `status`

এমুলেটার ইনস্টলেশনের বর্তমান অবস্থা রিপোর্ট করে।

```bash
bun run status
```

আউটপুট ফরম্যাট মেশিন-রিডেবল:

```
emulator=present
emulator_version=5541e5c
emulator_path=/Users/you/github/not-claude-code-emulator
install_mode=local-folder
support=supported
```

এক্সিট কোড 0 মানে সুস্থ। এক্সিট কোড 1 মানে কিছু মনোযোগ প্রয়োজন।

### `doctor`

বর্তমান অবস্থার উপর ভিত্তি করে ডায়াগনোস্টিক নির্দেশনা প্রদান করে।

```bash
bun run doctor
```

এটি ফাইল পরীক্ষা করে এবং কার্যকরী পরবর্তী ধাপ রিপোর্ট করে। এটি কিছু ইনস্টল, প্যাচ বা পরিবর্তন করে না। এটি শুধু পড়ে এবং রিপোর্ট করে।

### `patch_apply`

টার্গেটে প্যাচ প্রয়োগ করে (বর্তমানে ভবিষ্যৎ এক্সটেনশনের জন্য একটি স্টাব)।

```bash
bun run patch:apply
```

বর্তমান সংস্করণে, এটি পরিবেশ যাচাই করে কিন্তু কোনো পিয়ার স্টেট পরিবর্তন করে না। ভবিষ্যৎ সংস্করণগুলোতে রোলব্যাক মার্কার সহ আসল প্যাচিং বাস্তবায়ন হতে পারে।

### `patch_revert`

পূর্বে প্রয়োগ করা প্যাচ প্রত্যাবর্তন করে (বর্তমানে ভবিষ্যৎ এক্সটেনশনের জন্য একটি স্টাব)।

```bash
bun run patch:revert
```

বর্তমান সংস্করণে, এটি পরিবেশ যাচাই করে কিন্তু কোনো পিয়ার স্টেট পরিবর্তন করে না। ভবিষ্যৎ সংস্করণগুলোতে রোলব্যাক মার্কার ব্যবহার করে আসল প্রত্যাবর্তন বাস্তবায়ন হতে পারে।

## কেন স্বয়ংক্রিয় হুক শুধুমাত্র যাচাই-মাত্র

এই প্লাগইনে স্বয়ংক্রিয় হুক শুধুমাত্র যাচাই এবং মেটাডেটায় সীমাবদ্ধ। এগুলো স্বয়ংক্রিয়ভাবে প্যাচ প্রয়োগ করে না কারণ:

1. স্পষ্ট ব্যবহারকারীর ইচ্ছা ছাড়া একটি পিয়ার পরিবর্তন করা ন্যূনতম আশ্চর্যের নীতি লঙ্ঘন করে
2. প্যাচিং ব্যর্থতার জন্য মানব পর্যালোচনা প্রয়োজন, নীরব পুনরায় চেষ্টা নয়
3. প্রত্যাবর্তন অবস্থা পুনরুদ্ধারের জন্য স্পষ্ত সম্মতি প্রয়োজন

হুকগুলো যখন বিচ্যুতি সনাক্ত হয় তখন সতর্ক করে। আপনি প্রয়োগ, প্রত্যাবর্তন, বা পরিবেশ অপরিবর্তিত রাখার সিদ্ধান্ত নেন।

| Windows  | সমর্থিত | ড্রাইভ-অক্ষর এবং ব্যাকস্ল্যাশ-ভিত্তিক প্লাগইন আবিষ্কার সমর্থন করে |

## সামঞ্জস্য ক্যানারি

পিন করা টার্গেটের বিরুদ্ধে আপস্ট্রিম বিচ্যুতি যাচাই করতে:

```bash
bun run compat:canary
```

এটি একটি রিড-ওনলি চেক যা কিছু পরিবর্তন না করেই ফিক্সচার অখণ্ডতা এবং আপস্ট্রিম রেফারেন্স যাচাই করে। এটি পিন করা সমর্থিত টার্গেটে 0 এ প্রস্থান করে।
