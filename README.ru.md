# openclaw-cc-camouflage

Вспомогательный плагин обслуживания для OpenClaw, который помогает проверять статус not-claude-code-emulator. Этот пакет не является форком апстрим-проектов. Он предоставляет явные инструменты без автоматических хуков.

## Что это

`openclaw-cc-camouflage` — это плагин обслуживания, который:

- Проверяет наличие и состояние эмулятора перед любыми операциями
- Сообщает статус и предоставляет диагностические рекомендации
- Предоставляет заглушки реализации для будущих операций патчинга

Он не применяет патчи автоматически во время установки. Все мутации требуют явного вызова инструмента.

## Предварительные требования и порядок установки

Порядок установки имеет значение. У вас должно быть следующее в наличии, прежде чем этот плагин сможет работать:

1. **`not-claude-code-emulator`** (коммит `5541e5c`)
   - Среда выполнения сообщений, предоставляющая совместимые с Anthropic интерфейсы
   - Установка через npm: `npm install -g not-claude-code-emulator`
   - Или клонирование в `~/github/not-claude-code-emulator`

2. **`openclaw-cc-camouflage`** (этот пакет)
   - Устанавливать последним, после появления эмулятора

Настройте переменную окружения:

```bash
export OC_CAMOUFLAGE_EMULATOR_ROOT="$HOME/github/not-claude-code-emulator"
```

Или используйте резервные пути:

```bash
export OC_CAMOUFLAGE_EMULATOR_FALLBACK_PATHS="/opt/emulator:$HOME/.local/share/emulator"
```

## Доступные инструменты

Этот плагин предоставляет четыре явных инструмента. Они не являются автоматическими хуками.

### `status`

Сообщает текущее состояние установки эмулятора.

```bash
bun run status
```

Формат вывода читается машиной:

```
emulator=present
emulator_version=5541e5c
emulator_path=/Users/you/github/not-claude-code-emulator
install_mode=local-folder
support=supported
```

Код выхода 0 означает исправность. Код выхода 1 означает, что что-то требует внимания.

### `doctor`

Предоставляет диагностические рекомендации на основе текущего состояния.

```bash
bun run doctor
```

Это проверяет файлы и сообщает о выполнимых следующих шагах. Оно не устанавливает, не патчит и не модифицирует ничего. Только читает и сообщает.

### `patch_apply`

Применяет патчи к цели (в настоящее время заглушка для будущего расширения).

```bash
bun run patch:apply
```

В текущей версии это проверяет окружение, но не модифицирует состояние пира. Будущие версии могут реализовать фактический патчинг с маркерами отката.

### `patch_revert`

Откатывает ранее примененные патчи (в настоящее время заглушка для будущего расширения).

```bash
bun run patch:revert
```

В текущей версии это проверяет окружение, но не модифицирует состояние пира. Будущие версии могут реализовать фактический откат с использованием маркеров отката.

## Почему автоматические хуки предназначены только для проверки

Автоматические хуки в этом плагине ограничены только проверкой и метаданными. Они не применяют патчи автоматически, потому что:

1. Мутация пира без явного намерения пользователя нарушает принцип наименьшего удивления
2. Сбои патчинга требуют человеческого обзора, а не беззвучных повторных попыток
3. Откат требует явного согласия для восстановления состояния

Хуки предупреждают, когда обнаруживается отклонение. Вы решаете, применять, откатывать или оставлять окружение без изменений.

## Поддержка платформ

| Платформа | Статус | Примечания |
|----------|--------|-------|
| macOS    | Поддерживается | Основная среда рабочего стола |
| Linux    | Поддерживается | Те же закрепленные апстрим-фикстуры |
| Windows  | Поддерживается | Поддерживает обнаружение плагинов на основе буквы диска и обратной косой черты |

## Канарейка совместимости

Чтобы проверить апстрим-отклонение относительно закрепленных целей:

```bash
bun run compat:canary
```

Это проверка только для чтения, которая проверяет целостность фикстур и апстрим-ссылки без модификации чего-либо. Она завершается с кодом 0 на закрепленных поддерживаемых целях.

```bash
# Установить зависимости
bun install

# Проверка типов
bun run typecheck

# Запустить тесты
bun run test:unit
bun run test:integration

# Проверить патчи относительно фикстур
bun run verify:patches

# Проверить безопасность публикации
bun run check:publish-safety
```
